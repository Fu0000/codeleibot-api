# OpenFang Twitter Hand → 当前系统落地映射（V1）

## 目标

在不切换主系统的前提下，借鉴 OpenFang 的 Twitter Hand 机制，升级现有 Twitter 作战台（https://twitter.hichuhai.com），形成可持续增长流水线。

---

## 1) 核心映射表

### 1.1 配置层

| OpenFang 字段 | 含义 | 当前系统落地建议 |
|---|---|---|
| `twitter_style` | 内容语气风格 | 增加风格枚举：专业/口语/犀利/教育/故事 |
| `post_frequency` | 发布频率 | 增加排期模板：1/3/5 daily |
| `content_topics` | 内容主题池 | 增加主题池配置并打标签 |
| `brand_voice` | 人设语气描述 | 增加“人设说明”字段作为生成前缀 |
| `thread_mode` | 是否包含线程 | 在生成器中增加 thread 开关 |
| `approval_mode` | 审批发布模式 | 增加“待审核队列”，默认人工确认后发布 |
| `content_queue_size` | 队列容量 | 增加草稿队列上限（默认10） |
| `engagement_hours` | 互动时段 | 增加提醒窗口（仅在时段内执行互动任务） |

### 1.2 流程层

| OpenFang 阶段 | 当前系统实现建议 |
|---|---|
| API 初始化/状态恢复 | 启动时读取本地配置 + 草稿队列 + 发布记录 |
| 策略与排期创建 | 依据频率模板自动生成日程建议 |
| 研究与趋势输入 | 手动或半自动导入热点，作为选题上下文 |
| 内容生成 | 继续使用现有 Hook/Thread 生成器 |
| 审批与发布 | MVP阶段人工复制发布；后续接 API 自动发布 |
| 绩效追踪 | 每日记录曝光/互动/转粉；每周复盘 |

---

## 2) 先做哪些（高 ROI）

1. **审批队列（必须）**
   - 所有内容先进队列，人工一键通过后发布
2. **频率模板（必须）**
   - 1/3/5 daily 三种模板
3. **人设语气与主题池（必须）**
   - 保证内容一致性
4. **复盘表自动填充（高优）**
   - 每日记录核心三指标

---

## 3) 暂不引入（避免分心）

- 全量迁移 OpenFang
- 全自动互动（自动回复/自动点赞）
- 重型多代理编排

原因：当前主目标是增粉效率，不是平台替换。

---

## 4) 验证标准（保持清醒）

每个借鉴点必须满足：

1. **可验证**：能用一周数据证明有效
2. **可回滚**：功能可开关，不影响主流程
3. **低耦合**：不绑定单一平台架构

若连续两周无提升，立即降级或删除该机制。

---

## 5) 执行顺序（建议）

- 第1步：审批队列 + 频率模板
- 第2步：主题池 + 人设语气统一
- 第3步：复盘指标自动化
- 第4步：再评估是否引入更多自动化

---

## 6) 结论

- 方向：**借鉴 OpenFang 的机制，不照搬系统**
- 原则：**先验证再扩展，始终服务于 10万粉目标**
