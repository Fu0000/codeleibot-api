# Twitter 10万粉 + 桌面情感宠物 项目规划（V1）

## 0. 项目目标

- **主目标A（增长）**：在 X/Twitter 打造可持续增长系统，冲刺 100,000 粉丝。
- **主目标B（产品）**：同步打造「桌面智能情感宠物」MVP（可对话、可表达、可动作），作为内容与品牌护城河。
- **策略**：内容增长与硬件研发并行推进，形成“内容反哺产品、产品反哺内容”的复利循环。

---

## 1. 项目范围

### 1.1 增长侧（Twitter）

1. 人设与内容支柱确定（创始人叙事 + AI实战 + 产品共创）
2. 内容生产流水线（Hook -> Thread -> 发布 -> 复盘）
3. 运营节奏与指标看板（曝光、互动、转粉率）
4. 高频 A/B 测试（标题、开头、CTA、发布时间）

### 1.2 产品侧（桌面情感宠物）

1. 形象系统（已选 A：治愈萌系）
2. 工业设计草案（三视图 -> 分件 -> 3D 打印）
3. 硬件最小可用链路（ESP32 + 屏幕 + 麦克风 + 喇叭 + 舵机）
4. 软件映射（AIRI 情绪状态 -> 动作/灯光/表情）

---

## 2. 里程碑（建议 8 周）

### M1（第1-2周）：增长系统上线 + 形象定版

- Twitter：
  - 确定账号定位与 3 大内容支柱
  - 每日发布节奏跑通（2短推 + 1Thread）
  - 完成第一版数据看板
- 硬件：
  - A 方案三视图确认
  - 第一版外壳结构参数冻结
  - 打印首个低精度样件

### M2（第3-4周）：内容爆款迭代 + 硬件最小演示

- Twitter：
  - 形成 5 套可复用爆款模板
  - 每周复盘并保留 Top 10 模板
- 硬件：
  - 跑通语音输入输出（听/说）
  - 跑通 3 种情绪动作（开心/待机/撒娇）

### M3（第5-6周）：产品叙事绑定增长

- Twitter：
  - 公开连载《从0到1做情感宠物》
  - 发布制作日志、失败复盘、功能投票
- 硬件：
  - 外壳 V2（可装配）
  - 稳定运行 30 分钟 demo

### M4（第7-8周）：规模化增长验证

- Twitter：
  - 打通选题池 -> 自动生成 -> 人工精修 -> 排期
  - 明确月度增长目标与预算（若投放）
- 硬件：
  - 可演示版本（可拍视频、可持续迭代）
  - 明确下一版 BOM 与成本区间

## 5. 成功指标（KPI）

### 5.1 增长侧 KPI

- 周发布数 ≥ 21 条（含 Thread）
- 周净增粉丝（先从可持续曲线开始）
- Thread 平均互动率持续提升
- 每周至少 2 条高保存/高转发内容

### 5.2 产品侧 KPI

- 2 周内完成首个可装配壳体样件
- 4 周内完成“可听可说可动”MVP
- 8 周内形成稳定可拍摄 demo

---

## 6. 风险与对策

1. **风险**：增长与研发互相抢时间
   - 对策：固定内容生产时段，硬件开发分段推进
2. **风险**：形象频繁改动导致返工
   - 对策：先冻结 V1 外观参数，迭代走版本管理
3. **风险**：硬件调试周期不可控
   - 对策：先面包板验证，再上壳体；先最小功能再扩展
