# Twitter 10万粉 + 桌面情感宠物 项目规划（V1）

## 0. 项目目标

- **主目标A（增长）**：在 X/Twitter 打造可持续增长系统，冲刺 100,000 粉丝。
- **主目标B（产品）**：同步打造「桌面智能情感宠物」MVP（可对话、可表达、可动作），作为内容与品牌护城河。
- **策略**：内容增长与硬件研发并行推进，形成“内容反哺产品、产品反哺内容”的复利循环。

---

## 1. 项目范围

### 1.1 增长侧（Twitter）

1. 人设与内容支柱确定（创始人叙事 + AI实战 + 产品共创）
2. 内容生产流水线（Hook -> Thread -> 发布 -> 复盘）
3. 运营节奏与指标看板（曝光、互动、转粉率）
4. 高频 A/B 测试（标题、开头、CTA、发布时间）

### 1.2 产品侧（桌面情感宠物）

1. 形象系统（已选 A：治愈萌系）
2. 工业设计草案（三视图 -> 分件 -> 3D 打印）
3. 硬件最小可用链路（ESP32 + 屏幕 + 麦克风 + 喇叭 + 舵机）
4. 软件映射（AIRI 情绪状态 -> 动作/灯光/表情）

## 2. 里程碑（建议 8 周）

### M1（第1-2周）：增长系统上线 + 形象定版

- Twitter：
  - 确定账号定位与 3 大内容支柱
  - 每日发布节奏跑通（2短推 + 1Thread）
  - 完成第一版数据看板
- 硬件：
  - A 方案三视图确认
  - 第一版外壳结构参数冻结
  - 打印首个低精度样件

### M2（第3-4周）：内容爆款迭代 + 硬件最小演示

- Twitter：
  - 形成 5 套可复用爆款模板
  - 每周复盘并保留 Top 10 模板
- 硬件：
  - 跑通语音输入输出（听/说）
  - 跑通 3 种情绪动作（开心/待机/撒娇）

### M3（第5-6周）：产品叙事绑定增长

- Twitter：
  - 公开连载《从0到1做情感宠物》
  - 发布制作日志、失败复盘、功能投票
- 硬件：
  - 外壳 V2（可装配）
  - 稳定运行 30 分钟 demo

### M4（第7-8周）：规模化增长验证

- Twitter：
  - 打通选题池 -> 自动生成 -> 人工精修 -> 排期
  - 明确月度增长目标与预算（若投放）
- 硬件：
  - 可演示版本（可拍视频、可持续迭代）
  - 明确下一版 BOM 与成本区间

## 8. 今日可直接发布文案（Day 1）

### 短推1（09:00）

大多数人做Twitter做不起来，不是因为不努力。  
是因为把“发内容”当目标，而不是把“增长系统”当目标。  
**系统 > 灵感。节奏 > 情绪。**

### 短推2（13:00）

我现在只做这4步：  
1) 选题（有需求+有争议+可落地）  
2) 结构（Hook→Proof→Action）  
3) 分发（1-3-7复用）  
4) 复盘（只看曝光/互动/转粉）

这4步跑顺，账号自然涨。

### Thread（21:00）

1/10 如果你想把 Twitter 做成增长引擎，不要再“随缘发内容”了。  
我用一套可复制的系统，把内容变成稳定产出机器（附完整框架）。

2/10 先说结论：  
做账号增长，最重要的不是文笔，而是**流程设计**。

3/10 我把流程压成4步：  
选题 → 结构 → 分发 → 复盘  
每一步都可量化、可迭代。

4/10 ① 选题规则：  
只做“有搜索需求 + 有争议 + 可执行”的交集。  
没有这三点，流量很难稳定。

5/10 ② 结构模板：  
Hook（抓注意）→ Proof（给证据）→ Action（给动作）  
每条内容只讲一个核心观点。

6/10 ③ 分发机制：  
1-3-7复用法。  
当天主贴，3天后改写，7天后复盘再发。

7/10 ④ 复盘指标：  
我只看3个：曝光、互动率、转粉率。  
不看这3个，优化就是盲飞。

8/10 这套方法特别适合：  
有专业能力，但输出不稳定的创业者/创作者。

9/10 你不需要更努力，你需要更稳定的系统。  
系统会帮你对抗拖延和情绪波动。

10/10 如果你要，我下一条发：  
《我正在用的7天内容排期模板（可直接套用）》  
关注我，评论“模板”。
