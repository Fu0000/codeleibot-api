### 一、核心洞察（高ROI）

1. **选“用户已熟悉的认知锚点”比原创世界观更容易起量**
   - 案例中选蜡笔小新，不需教育市场，用户一眼识别。
   - 启发：做内容增长，先借“强识别符号”拿到首波停留，再谈品牌独特性。

2. **提示词不是“描述”，而是“生产规格书”**
   - 其提示词包含：角色、风格质感、结构、文本准确性、输出比例。
   - 启发：高稳定内容生产要从“灵感驱动”转到“约束驱动”。

3. **短链路转化：停留 -> 读完 -> 互动**
   - 条漫天然有阅读停留时长优势，平台更愿意放量。
   - 启发：我们应优先设计“必须读完才懂”的内容结构（而非一眼滑过）。

4. **格式一致性是账号资产，不只是审美问题**
   - 两格 + 3:4 固定输出，形成账号可识别风格。
   - 启发：固定内容骨架可显著提升批量化和用户记忆点。

5. **爆量不是偶然，核心是可复用系统**
   - 先用高命中赛道跑通，再用低成本批量生产放大。
   - 启发：我们的 war-room 要优先支持“模板复用 + 复盘迭代”而非单次神作。

### 二、可迁移到我们项目的执行改造

#### 1) Prompt Spec Builder（提示词规格化）
在现有 war-room 增加“规格化提示词模块”：
- 角色锁定（你是谁）
- 风格约束（质感关键词）
- 结构约束（长度、段数、格式）
- 文本准确性规则（不可错字/固定术语）
- 输出格式（平台比例/长度）

#### 2) 强识别内容模板库（不直接蹭IP，借“认知框架”）
- 模板方向：
  - 反差两段式（A场景 -> B反转）
  - 固定三段式（结论 -> 证据 -> 动作）
  - 角色视角连续剧（创始人战时报）
- 原则：学习结构与机制，不直接复制受版权保护角色。

#### 3) “停留优先”内容策略
- 文案中嵌入必须读完才能闭环的信息设计：
  - 开头抛矛盾
  - 中段给方法
  - 尾段给可执行动作
- 对应KPI新增：平均阅读完成率（可用互动率近似）

### 三、一周A/B实验（可直接执行）

- **实验目标**：提升互动率与转粉效率
- **样本周期**：7天
- **变量设计**：
  - A组：自由写法（当前基线）
  - B组：规格化模板写法（角色+质感+结构+CTA固定）
- **对比指标**：曝光、互动率、转粉、收藏/转发（能取则取）
- **判定阈值**：
  - 若B组互动率连续3天领先A组 >= 20%，则进入默认模板

3. **平台波动风险**：单平台爆量不等于长期稳定，必须复盘连续7-14天数据。
